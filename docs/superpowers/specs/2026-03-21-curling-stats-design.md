# Curling Stats — Дизайн-документ

## Обзор
PWA-приложение для ведения статистики бросков в керлинге в реальном времени. Статистик на скамейке быстро вносит оценки бросков обеих команд, приложение автоматически считает аналитику.

## Стек
- **Frontend:** React (PWA, Service Worker для офлайна)
- **Backend:** Node.js + Express
- **БД:** SQLite (файловая, без отдельного сервера)
- **Хостинг:** свой сервер
- **Авторизация:** нет (MVP)

## Модель данных

```
Game (id, date, team_home, team_away, 
      color_home, color_away,
      hammer_first_end: home|away,
      max_ends: 10,
      status: active|finished)

End (id, game_id, number: 1-10+,
     score_home, score_away,
     hammer: home|away)

Shot (id, game_id, end_number,
      shot_number: 1-16,
      team: home|away,
      player_number: 1-4,
      type: draw|takeout|null,
      turn: inturn|outturn|null,
      score: 0|25|50|75|100|null,
      is_throwaway: boolean)
```

### Правила хаммера
- Хаммер в 1-м энде задаётся при создании игры
- Команда с хаммером бросает вторым в каждой паре
- Хаммер переходит к проигравшей энд команде
- При нулевом энде (0:0) хаммер остаётся

## Порядок бросков в энде

16 бросков, чередуясь между командами. Каждый игрок — 2 камня подряд (один от каждой команды):

| Бросок | Команда       | Игрок |
|--------|---------------|-------|
| 1      | Без хаммера   | Lead (1) |
| 2      | С хаммером    | Lead (1) |
| 3      | Без хаммера   | Lead (1) |
| 4      | С хаммером    | Lead (1) |
| 5      | Без хаммера   | Second (2) |
| 6      | С хаммером    | Second (2) |
| 7      | Без хаммера   | Second (2) |
| 8      | С хаммером    | Second (2) |
| 9      | Без хаммера   | Third (3) |
| 10     | С хаммером    | Third (3) |
| 11     | Без хаммера   | Third (3) |
| 12     | С хаммером    | Third (3) |
| 13     | Без хаммера   | Skip (4) |
| 14     | С хаммером    | Skip (4) |
| 15     | Без хаммера   | Skip (4) |
| 16     | С хаммером    | Skip (4) |

## Экраны

### Экран 1 — Список игр (главная)
- Список всех игр: дата, команды, счёт, статус
- Кнопка «Новая игра»
- Тап по игре → открыть (если активна) или аналитику (если завершена)

### Экран 2 — Создание игры
- Название нашей команды
- Название команды противника
- Цвет камней нашей команды (выбор)
- Цвет камней противника (выбор)
- Кто имеет хаммер в 1-м энде
- Макс. эндов (по умолчанию 10)
- Кнопка «Начать игру»

### Экран 3 — Ввод бросков (основной рабочий экран)
**Шапка:**
- Табло счёта (цвета команд)
- Номер энда
- Автоматически: какой бросок (1-16), какая команда, какой игрок

**Ввод (3 тапа + подтверждение):**
1. Тип броска: **Draw** / **Takeout** — 2 кнопки
2. Вращение: **Inturn** / **Outturn** — 2 кнопки
3. Оценка: **0** / **25** / **50** / **75** / **100** — 5 кнопок
4. Кнопка **«Следующий бросок →»**

**Или:**
- Кнопка **«Проброс»** — бросок записывается, но исключается из статистики. Тип/вращение/оценка не вводятся.

**Навигация:**
- Кнопка **«← Назад»** — вернуться к предыдущему броску, исправить
- После 16-го броска → ввод результата энда (кто взял, сколько камней)
- После последнего энда или кнопка «Завершить игру» → переход к аналитике

### Экран 4 — Аналитика

**Вкладка 1: Сравнение команд**
Таблица — каждый наш игрок vs соответствующий номер соперника:

| Позиция | Наш | Соперник | Наш Draw | Сопер. Draw | Наш Take | Сопер. Take |
|---------|-----|----------|----------|-------------|----------|-------------|
| Lead    | 68% | 72%      | 75%      | 69%         | 62%      | 75%         |
| Second  | ... | ...      | ...      | ...         | ...      | ...         |
| Third   | ... | ...      | ...      | ...         | ...      | ...         |
| Skip    | ... | ...      | ...      | ...         | ...      | ...         |
| **Команда** | **X%** | **Y%** | ... | ... | ... | ... |

Итоговый счёт игры.

**Вкладка 2: Глубокий анализ (только наша команда)**
По каждому игроку:
- Inturn общее / Outturn общее
- Inturn Draw / Inturn Takeout
- Outturn Draw / Outturn Takeout

## API Endpoints

```
GET    /api/games              — список игр
POST   /api/games              — создать игру
GET    /api/games/:id          — игра с эндами и бросками
POST   /api/games/:id/shots    — записать бросок
PUT    /api/games/:id/shots/:n — исправить бросок
POST   /api/games/:id/ends     — записать результат энда
PUT    /api/games/:id/status   — завершить игру
GET    /api/games/:id/stats    — аналитика по игре
```

## Ограничения MVP
- Без авторизации
- Один тип игры (классический керлинг, 4 игрока)
- Типы бросков: draw / takeout (расширяем позже)
- Нет турниров/сезонов
- Нет экспорта данных
